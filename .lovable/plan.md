

# Remap Comment Aggregation Logic in Inspection Summary

## Problem

The `generateSummaryFromInspection` function currently maps inspection line item comments into only two summary sections:

| Status | Current Destination | Required Destination |
|---|---|---|
| `fail` | Critical Actions | Critical Actions (no change) |
| `pass w/provisions` | Repairs, Alterations | **Future Considerations** |
| `pass` | (ignored) | **Repairs, Alterations** |

Comments from items with "pass" status that have remarks are silently dropped, and "pass w/provisions" items are incorrectly categorized.

## Solution

Update `generateSummaryFromInspection` and `handleRegenerateSummary` in `src/pages/InspectionForm.tsx` to implement the three-way mapping, plus update the real-time signature tracker to also watch "pass" items with comments.

### Changes (single file: `src/pages/InspectionForm.tsx`)

**1. `generateSummaryFromInspection` (~lines 226-315)**

Add a third accumulator array `futureConsiderations` and remap:

```text
Before:
  fail           -> criticalActions
  pass w/provisions -> repairsPerformed

After:
  fail              -> criticalActions       (unchanged)
  pass w/provisions -> futureConsiderations  (moved)
  pass (with comments) -> repairsPerformed   (new)
```

For each of the three item processors (equipment, systems, ziplines):
- `fail` items continue to push to `criticalActions`
- `pass w/provisions` items now push to `futureConsiderations` instead of `repairsPerformed`
- `pass` items with non-empty comments push to `repairsPerformed`

The function returns a third field: `futureConsiderations`.

**2. `handleRegenerateSummary` (~lines 507-526)**

Update to also set `future_considerations` from the new return value:

```typescript
setSummary(prev => ({
  ...prev,
  critical_actions: convertCircleBulletsToHtml(autoGenerated.criticalActions),
  repairs_performed: convertCircleBulletsToHtml(autoGenerated.repairsPerformed),
  future_considerations: convertCircleBulletsToHtml(autoGenerated.futureConsiderations),
}));
```

**3. Auto-populate on load (~lines 476-504)**

Add the same conditional population for `future_considerations`:

```typescript
if (!prev.future_considerations?.trim() && autoGenerated.futureConsiderations) {
  newSummary.future_considerations = convertCircleBulletsToHtml(autoGenerated.futureConsiderations);
}
```

**4. Real-time signature tracker (~lines 528-606)**

Expand `getFailProvisionsSignature` to also include "pass" items that have non-empty comments, so edits to passing items with remarks trigger a silent re-aggregation. The signature will capture all three statuses that feed into summary sections.

## Atomicity and Concurrency

- The aggregation runs synchronously from in-memory React state -- no async race conditions
- The existing `isInternalUpdateRef` flag prevents the auto-save watcher from double-firing during regeneration
- The 800ms debounce on the real-time tracker coalesces rapid edits into a single regeneration pass
- No database writes occur during aggregation; it only updates local state which then flows through the standard save pipeline

## Files Changed

1. **`src/pages/InspectionForm.tsx`** -- Update `generateSummaryFromInspection`, `handleRegenerateSummary`, auto-populate effect, and real-time signature tracker

## No Other Files Affected

- `SummarySection.tsx` already renders `future_considerations` with its own editor -- no changes needed
- Edge functions (`generate-inspection-html`, `generate-inspection-pdf`) already render `future_considerations` from the database -- no changes needed
- Database schema already has the `future_considerations` column -- no migration needed
