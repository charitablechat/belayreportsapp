

# Plan: Add SMS/Text Sharing to Report Viewer (v2.2.92)

## Overview

This plan adds a dedicated **SMS/Text** sharing button to the `HtmlReportViewer` header, providing a direct path to share report metadata via text message. The button will appear alongside the existing Email, Download, and Share buttons, following the **Minimal Brutalism** design aesthetic shown in the reference screenshot.

## Current State

The `HtmlReportViewer` already has these buttons in the top-right header:
- Email (envelope icon) - sends full HTML via edge function
- Download (download icon) - saves HTML file locally  
- Share (share icon) - triggers Web Share API
- Close (X icon)

## Target Implementation

Add an **SMS/Text** button (message-square icon) that:
1. Opens the native SMS/messaging app with pre-filled content
2. Includes report metadata (type, organization, date) in the message
3. Only appears on mobile devices (SMS links don't work reliably on desktop)
4. Uses the same high-contrast "Minimal Brutalism" styling as the Email button

## Technical Details

### SMS URI Format

Standard SMS URI that works across platforms:
```
sms:?body=Report%20Information...
```

**Message Template** (under 200 characters for SMS compatibility):
```
[Report Type] Report
Organization: [Name]
Date: [Date]
Generated by Rope Works Inspection System
```

### Code Snippets

**SMS Link Generation:**
```typescript
export function generateSmsLink(
  reportType: string,
  organization: string,
  date: string
): string {
  const message = [
    `${formatReportType(reportType)} Report`,
    `Organization: ${organization}`,
    `Date: ${date}`,
    `Generated by Rope Works Inspection System`
  ].join('\n');
  
  // Use sms:?body= format (no recipient)
  return `sms:?body=${encodeURIComponent(message)}`;
}
```

## Visibility Logic

The SMS button only appears when:
1. The device is detected as mobile (`isMobile()` from mobile-detection)
2. The viewer is open (report has been generated)
3. Report was successfully generated (viewer only opens after completion + generation)

The existing flow already ensures reports are only viewable after completion:
1. "Complete" button must be clicked first (sets `status: 'completed'`)
2. "Generate Report" button only appears when `status === 'completed'`
3. `HtmlReportViewer` only opens after successful HTML generation

## Platform Support

| Platform | SMS Button | Behavior |
|----------|------------|----------|
| iOS Safari/PWA | Visible | Opens Messages with pre-filled text |
| Android Chrome/PWA | Visible | Opens default messaging app |
| Desktop browsers | Hidden | SMS links unreliable on desktop |

## Security Considerations

- No sensitive data in SMS message (only metadata: type, org, date)
- No backend URLs or secrets exposed
- Report content stays on-device
